    Visual Studio 2012 环境配置¶

    OpenCV 环境¶

    一、配置 windows 环境变量¶

    加入以下两行 

    ...\opencv\build\x64\vc11\bin
...\opencv\build\x86\vc11\bin
    PS : VS2012 用 vc11，VS2013 用 vc12

    二、vs 环境¶

    1. 新建一个项目
    2. 进入“属性管理器”选项卡后，找到项目中的Microsoft.Cpp.Win32.user,右键选择属性。 注意：图中所示的Debug和Release两种模式，不影响最终配置效果，选择其中一种就行
    3. 紧接上步，打开新的页面，找到“通用属性”--“VC++目录”，在右侧找到“包含目录(include files)”，加入如下三条信息，都是opencv目录下的指定目录：
    ...\opencv\build\include\opencv2
...\opencv\build\include\opencv
...\opencv\build\include
    1. 还是同一页面，“通用属性”--“VC++目录”，在右侧找到“库目录(library files)”，加入如下两条信息：
    ...\opencv\build\x86\vc11\lib
...\opencv\build\x64\vc11\lib

    注意：VS2012对应的需要Opencv的vc11路径，不可选错

    三、依赖项¶

    仍旧是同一页面，找到：“通用属性”--“链接器”--“输入”，在右侧找到“附加依赖项”，加入如下所有信息，该信息对应在opencv的：

    ...\opencv\build\x64\vc11\lib

    Kinect 环境¶

    一、vs 环境¶

    1. 新建一个项目
    2. 进入“属性管理器”选项卡后，找到项目中的Microsoft.Cpp.Win32.user,右键选择属性。 注意：图中所示的Debug和Release两种模式，不影响最终配置效果，选择其中一种就行
    3. 紧接上步，打开新的页面，找到“通用属性”--“VC++目录”，在右侧找到“包含目录(include files)”，加入如下三条信息: 
    C:\Program Files\Microsoft SDKs\Kinect\v2.0_1409\inc
    1. 还是同一页面，“通用属性”--“VC++目录”，在右侧找到“库目录(library files)”，加入如下两条信息： 
    C:\Program Files\Microsoft SDKs\Kinect\v2.0_1409\lib\x86
C:\Program Files\Microsoft SDKs\Kinect\v2.0_1409\lib\x64

    二、依赖项¶

    仍旧是同一页面，找到：“通用属性”--“链接器”--“输入”，在右侧找到“附加依赖项”，加入如下依赖：

    Kinect20.lib

    PS：该文件名可能会变化，具体请见 C:\Program Files\Microsoft SDKs\Kinect\v2.0_1409\Lib\x64 中的 lib 文件

    导出配置（复用）¶

    先在属性界面右击新建属性，将上述环境变量在该新建文件中配置成功后保存，以后想用这个配置的时候只需要添加这个属性文件。
